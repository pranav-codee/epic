# Teams Application Testing Guide — EPIC v1

This guide walks through deploying and testing the EPIC Teams app **end to end**, from package upload through a live Teams notification landing in a channel.

## Part 1 — Prerequisites

| Requirement | How to confirm |
|---|---|
| You are an EPL Microsoft 365 tenant admin (or have *Upload custom apps* permission) | Visit <https://admin.microsoft.com> → Settings → Org settings → Integrated apps |
| Microsoft Teams desktop or web client | <https://teams.microsoft.com> |
| Teams Developer Portal access (optional, recommended) | <https://dev.teams.microsoft.com> |
| The packaged `epic-teams-app/epic-teams-app.zip` from this repo | Generated by the build; do not modify after signing |
| A Teams channel where you'll receive ticket notifications | Any team channel you have permissions on |
| The EPIC backend running and reachable at the hostname declared in `manifest.json` | Default placeholder: `epic.epl.local` |

## Part 2 — One-time configuration

### 2.1 Create the Teams Incoming Webhook (for notifications)

1. Open Microsoft Teams.
2. Navigate to the channel that should receive ticket notifications (e.g. *IT Support → General*).
3. Click `⋯` (More options) next to the channel name → **Manage channel** → **Connectors** (or **Settings → Connectors** depending on the client version).
4. Find **Incoming Webhook** → **Add** → **Configure**.
5. Give it a name (e.g. `EPIC Notifications`), optionally upload an icon, click **Create**.
6. **Copy the webhook URL** — you only get to see it once.
7. Save it in `epic-backend/.env`:
   ```
   TEAMS_WEBHOOK_URL=https://epl.webhook.office.com/webhookb2/........@.../IncomingWebhook/........./........
   ```
8. Restart the backend so the new env var is picked up.

### 2.2 Confirm Teams notifications are turned on

In `epic-backend/.env`:
```
TEAMS_NOTIFICATIONS_ENABLED=true
```

### 2.3 (Optional) Verify the webhook independently

From any shell with `curl`:
```bash
curl -H "Content-Type: application/json" \
     -d '{"text":"EPIC webhook smoke test ✅"}' \
     "$TEAMS_WEBHOOK_URL"
```
You should see the message appear in the configured Teams channel within seconds. If you see `400 Bad Request` or no message, the webhook URL is incorrect — re-copy it from step 2.1.

## Part 3 — Upload the Teams app package

### 3.1 Sideload to your own account (fastest)

1. Open Teams → **Apps** (left rail).
2. Click **Manage your apps** → **Upload an app** → **Upload a custom app**.
3. Select `epic-teams-app/epic-teams-app.zip` from this repo.
4. Teams opens an install dialog: choose **Add** to install personally.

### 3.2 Distribute organisation-wide (for pilot / Phase 1)

1. Go to the [Teams admin centre](https://admin.teams.microsoft.com) → **Teams apps → Manage apps**.
2. Click **Upload new app** → upload `epic-teams-app.zip`.
3. Set **Org-wide app status** to **Allow**.
4. Optionally create an **App setup policy** that pins EPIC to the rail for all employees.

## Part 4 — Smoke test inside Teams

### 4.1 Open the personal app

1. In Teams left rail click `…` → search **EPIC** → click it.
2. The **My Tickets** tab loads — this is `https://epic.epl.local/employee` (or your real FQDN).
3. You'll be prompted to sign in with Microsoft. Complete the Microsoft Authenticator prompt on your phone.
4. After sign-in, you should land on the Employee Home screen.

### 4.2 Create a ticket from inside Teams

1. Click **New Ticket**.
2. Fill in:
   - Title: `Teams test — please ignore`
   - Description: `End-to-end smoke test from the Teams tab.`
   - Category: `OTHER`
   - Priority: `LOW`
3. Click **Submit ticket**. You should see the ticket detail page with a new `EPIC-YYYY-NNNNNN` number.

### 4.3 Confirm the notification arrived

1. Open the Teams channel from Part 2.1.
2. Within ~5 seconds, you should see a card titled **🆕 Ticket created — EPIC-YYYY-NNNNNN** with the title, status (`OPEN`), priority, category, and an **Open Ticket in EPIC** button.

### 4.4 Run the full lifecycle (engineer workflow)

In the Teams **Admin** tab (only functional if your account has `IT_ENGINEER`, `IT_MANAGER`, or `SYSTEM_ADMIN`):

1. Open the ticket from the queue.
2. **Assign** it to yourself → ✅ expect *🆕 Ticket assigned* card.
3. Click **IN_PROGRESS** → ✅ expect *✏️ Ticket updated* card.
4. Click **PENDING_USER** → ✅ another *Ticket updated* card.
5. Switch back to **Employee** tab → post a comment → ✅ expect *Ticket updated* card.
6. Back in Admin: click **IN_PROGRESS** (user responded) → **RESOLVED** → ✅ expect *✅ Ticket resolved*.
7. Click **CLOSED** → ✅ expect *🔒 Ticket closed*.
8. Optional reopen test: while ticket is RESOLVED (not CLOSED), click **IN_PROGRESS** — should succeed (the agreed reopen transition).

### 4.5 Verify the audit log

In the ticket detail page, the **History** section should show one row per action, including who did it and any old → new field values. Verify:
- A `CREATE` entry by the employee.
- An `ASSIGN` entry with `old_value: null, new_value: <assignee id>`.
- Multiple `STATUS_CHANGE` rows for each transition.
- A `COMMENT_ADDED` row.
- A `CLOSED` row at the end.

### 4.6 Verify failure handling

1. In `.env`, temporarily replace `TEAMS_WEBHOOK_URL` with `https://invalid.example.com/webhook` and restart.
2. Create another ticket — ticket creation must still succeed (NFR-5.4-2).
3. Sign in as SYSTEM_ADMIN → call `GET /api/v1/notifications` (via OpenAPI docs or the browser DevTools): you should see a row with `status: FAILED` and the HTTP error preserved in `error_message`.
4. Restore the real webhook URL and restart.

## Part 5 — Troubleshooting

| Symptom | Likely cause / fix |
|---|---|
| Teams shows "Couldn't load this app" | `validDomains` in manifest doesn't match the FQDN where the React app is served — update and re-zip. |
| Sign-in loop | Browser is blocking third-party cookies inside the Teams iframe. Use Edge/Chrome with third-party cookies allowed for `*.epl.local`, or implement Teams SSO (a v1.1 enhancement) using `webApplicationInfo` + `microsoftTeams.authentication.getAuthToken()`. |
| No notification appears | (a) `TEAMS_WEBHOOK_URL` is empty, (b) `TEAMS_NOTIFICATIONS_ENABLED=false`, (c) the webhook expired — re-create it. Check `GET /api/v1/notifications` for the recorded delivery status. |
| Admin tab shows 403 | The signed-in user does not have an engineer role. A SYSTEM_ADMIN can grant roles in Admin → Users. |
| Backend logs show "Invalid state" on a transition | The state machine refused an illegal status move. The `allowed_target_states` array on `GET /tickets/{id}` lists what's reachable from the current state. |

## Part 6 — Decommissioning a test app

If you sideloaded for testing:
1. Teams → Apps → **Manage your apps** → find EPIC → **Remove**.
2. Org-wide: Teams admin centre → **Manage apps** → find EPIC → **Block** or **Delete**.
